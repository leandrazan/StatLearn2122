Introduction

We now learn something about making nice data visualisations with the famous ggplot2 package. An equivalent of ggplot is also available in the plotnine library in Python, so you might already be familier with it.

We will again work with the mpg data from the first tutorial, which is already loaded for you.

The readings in this tutorial follow R for Data Science, section 3.

Basics

In base R, when you want to have a scatter plot of mpg’s variables engine displacement (displ) on the x-axis vs. hwy (highway miles per gallon) on the y-axis, you’d run 
plot(mpg$displ, mpg$hwy)
An equivalent plot in ggplot2 is made with 
ggplot(mpg) + 
  geom_point(aes( x = displ, y = hwy))

That looks fancier, right? Good looks isn’t the only advantage of ggplot, however. Once you get used to the syntax, it’s way easier and faster!

In ggplot2, you build your graph in several layers. If you have a data frame (or tibble) called df, the steps to create a plot are as follows:

  1. You create an empty plot providing your data df by calling ggplot(df)
  2. You type a + ( important : this has to come at the end of a line)
  3. You add so called geometries (e.g. points or lines) with the geom_***-functions, e.g. 
  • geom_point() for points
  • geom_line() for lines
  • geom_histogram for histograms
  • geom_boxplot for boxplots
  • … the list goes on .. You can type geom_ and look at the autocompletion to see it.
  1. All geom_***-functions have an argument called mapping, which specifies how the variables in df should be mapped to the visual properties of your geometry (also called geom). The mapping argument is always given in combination with aes(), which takes arguments such as
    • x: what variable is mapped to the x-axis
    • y: what variable is mapped to the y-axis
    • colour: what variable is mapped to the color of your geom
    • shape, size (for geom_point): what variable is mapped to the shape and size of the points
    • linetype (e.g. for geom_line()): what variable is mapped to the linetype of the lines
For example, if you want the points to be colour-coded depending on the class the respective observations belongs to, you can do this with 
ggplot(mpg) + 
  geom_point(aes( x = displ, y = hwy, color = class))

As you can see, a legend is added automatically (which base R’s plot function doesn’t do). Try to change the code below such that the shape of your points is mapped to the variable drv (the type of drive train such as front-wheel drive).

ggplot(mpg) + 
  geom_point(aes( x = displ, y = hwy))
ggplot(mpg) + 
  geom_point(aes( x = displ, y = hwy, shape = ))
ggplot(mpg) + 
  geom_point(aes( x = displ, y = hwy, shape = drv))

You can also map the color to the class and the shape to drive train - give it at try!

Histograms

Now we want to make a histogram of the hwy variable. Histograms only need a variable that is mapped to the x-axis. When only the default options are used, the base R hist function produces nicer histograms (in my opinion). But this is fixed quickly, uncomment the comment (i.e. delete the #) in the code below to see how!

ggplot(mpg)+ 
  geom_histogram( aes( x= hwy) , binwidth = 3 # ,  fill= "gray", color = "black" 
  )

If you want to plot the density rather than the frequencies, add y = ..density.. to aes(). A kernel density estimate can then be added with geom_density(). Try it out in the following lines.

ggplot(mpg)+ 
  geom_histogram( aes( x= hwy) , binwidth = 3  ,  fill= "gray", color = "black") 
ggplot(mpg)+ 
  geom_histogram( aes( x= hwy , y = ..density..) , binwidth = 3  ,  fill= "gray", color = "black") + 
  geom_density()
ggplot(mpg)+ 
  geom_histogram( aes( x= hwy , y = ..density..) , binwidth = 3  ,  fill= "gray", color = "black") + 
  geom_density(aes( x = hwy), size = .9)

Boxplots

Last but not least, we’ll have a look at boxplots. In geom_boxplot, you map a discrete variable to the x-axis and a continuous one to the y-axis:

ggplot(mpg)+ 
  geom_boxplot( aes( x= class, y = hwy))

Facets

Sometimes you want to have seperate panels for subsets of your data. This can be done with the functions * facet_wrap(~ y, nrow = n) : split by y into n rows * facet_grid( x ~ y) : Split by x (vertically) and y (horizontally). Here, . can be used instead of a variable if you want to split by only one variable into horizontally or vertically arranged subpanels.

Change the last line to facet_grid( class ~ .) and facet_wrap( ~ class) to see the differences.

ggplot(mpg)+ 
  geom_point(aes( x = displ, y = hwy)) + 
  facet_grid(. ~ class)
Now change the following code to split into class horizontally and drv vertically. 
ggplot(mpg)+ 
  geom_point(aes( x = displ, y = hwy)) 
ggplot(mpg)+ 
  geom_point(aes( x = displ, y = hwy)) + 
  facet_grid()
ggplot(mpg)+ 
  geom_point(aes( x = displ, y = hwy)) + 
  facet_grid(drv ~ class )

Plot labels and saving plots

Also very important: the labels of your plot. With the lab() function, you can add a title, change the labels of your axes or change the variable name displayed in the legend:

ggplot(mpg) + 
  geom_point(aes( x = displ, y = hwy, color = class))+ 
  labs( title = "Scatterplot of displacement vs. highway miles per gallon", 
        x = "Engine Displacement", 
        y = "Highway miles per gallon", 
        color = "Class of Car") + 
  theme( text = element_text(size = 16))  # increases the font size of your text elements (labels, title, ...)

If you want to save the last plot as pdf named ‘MyScatterplot’ to a folder named ‘Plots’ in your current working directory, you may run the following code (only run if you want to save it):

dir.create(paste0(getwd(), "/Plots"))  # creates the directory
ggsave(filename = paste0(getwd(), "/Plots/MyScatterplot.pdf"), plot = last_plot(), 
       device = "pdf", height = 10, width = 12 )
